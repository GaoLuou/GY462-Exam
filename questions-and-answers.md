### Q1：租约内/租约间折现率五步法是什么，为什么这样设计？
### Why is the Intra/Inter-Lease 5-Step Method structured this way?

**核心矛盾**：同一栋楼，不同时段的现金流，风险完全不同——不能用同一个折现率。

| 时段 | 风险来源 | 类比 | 折现率 |
|------|---------|------|-------|
| 租约内（intra-lease）| 租户违约风险（低概率） | 持有债券 | 低（如 8%） |
| 租约间（inter-lease）| 找不找到新租户？租多少？空置多久？| 持有股票 | 高（如 15%） |

**五步法逻辑推导（以第1-3年有租约，第4-6年无租约为例）：**

```
Step 1：第一段租约（已签），从各年折回"今天"
        → 用 intra-lease rate（低风险）

Step 2：第二段租约（未签），从各年折回"第二段开始那一刻"
        → 用 intra-lease rate（假设届时已签约，租约内低风险）

Step 3：将 Step 2 的结果从"第二段开始"折回"今天"
        → 用 inter-lease rate（今天到第二段开始，穿越不确定期，高风险）

Step 4：售价从卖出年份折回"今天"
        → 用 inter-lease rate（时间最远，不确定性最高）

Step 5：总价值 = Step1 + Step3 + Step4
```

**关键理解**：Step 2 + Step 3 合起来说的是：
> 「假设新租约签到了，在新租约内的现金流是低风险的（8%）；
> 但你今天要为"3年后能不能签到租约"这件事定价，这是高风险的（15%）。」

**为什么简单法（一个折现率）是错的？**
简单法既高估了签约期现金流的风险，又低估了空置期的风险——两个方向都错，净效果不可预测。

### Q2：为什么 DCF 比 Cap Rate 更准确？Cap Rate 的"近视性"是什么？

**近视性（Myopia）的精确含义**：Cap Rate 只使用 Year 1 的 NOI 做快照定价，完全看不到未来 NOI 的增长、空置风险、资本支出等动态变化。

```
V = NOI₁ / Cap Rate   ← 只看第1年，后面一概不管
```

**举例**：两栋楼，Year 1 NOI 都是 £100万，Cap Rate 都是 5%

| | Year 1 NOI | 之后增长 | Cap Rate 估值 | DCF 估值 |
|--|----------|---------|-------------|---------|
| A 楼 | £100万 | 每年 0% | £2,000万 | £2,000万 |
| B 楼 | £100万 | 每年 3% | £2,000万 | **远高于 £2,000万** |

Cap Rate 判断两栋楼一样值——这就是近视：看不出 B 楼未来递增现金流的价值。

**Cap Rate 近视于以下所有因素**（全部被它忽略）：
- NOI 增长率 g
- 空置期（vacancy）
- 未来大修资本支出
- 租约到期后重新招租风险
- 持有期长短

**Gordon 模型修正**：
```
V = NOI / (r - g)
```
把增长率 g 嵌入分母，增长越快 → 分母越小 → 价值越高。但仍假设 g 永远不变，仍是简化模型。

**DCF 最准确**：逐年建模每期实际 NOI，没有任何假设被藏在单一比率里。

### Q3：Going-in Cap Rate vs Going-out Cap Rate，为什么出口比入口高？

```
Going-in Cap Rate（购入时）：NOI₁ / 购入价
Going-out Cap Rate（出售时）：NOI_{n+1} / 售出价  ← 通常更高
```

**Going-out ≥ Going-in 的原因**：
1. **物业老化**：设备磨损，维修成本上升，价值下降
2. **不确定性增加**：距离今天越远，市场越难预测，投资者要求更高回报
3. **折旧与过时**：建筑技术和审美会变化

> 实务中如果 Going-out < Going-in（"yield compression"），意味着你预测物业会增值——这是乐观假设，需要充分理由支撑。

### Q4：等收益率法（Equivalent Yield）公式是怎么推导的？各变量是什么意思？
### How is the Equivalent Yield formula derived?

**背景**：英国商业地产租约期间不能随意调价，当前租金（Passing Rent）往往不等于市场租金（ERV）。等收益率法专门处理"两段现金流"的估值。

**变量含义**：
- **I（Passing Rent）**：当前合同租金（可能低于或高于市场价）
- **R（ERV，Estimated Rental Value）**：市场估计租金（下次审查后应收的价格）
- **y（Equivalent Yield）**：等收益率，整体折现率
- **n**：距离下次租约审查/到期的年数

**现金流结构**：
```
第 0 → n 年：收租金 I（固定，按合同）
第 n 年之后：收租金 R（永续，恢复市场价）
```

**推导（拆成两部分）**：

*Part 1：Term — n 年收 I 的年金*
```
PV(Term) = I × [1 - (1+y)^(-n)] / y
```

*Part 2：Reversion — 第 n 年后收 R 的永续年金*
```
PV(Reversion) = R/y × (1+y)^(-n)
```
（先求第 n 年末的永续年金价值 R/y，再折现 n 期到今天）

*两部分相加并化简*：
```
V = I × [1-(1+y)^(-n)] / y  +  R/y × (1+y)^(-n)
  = I/y − I/[y×(1+y)^n]  +  R/[y×(1+y)^n]
  = I/y + (R − I) / [y × (1+y)^n]
```

**直觉理解**：
- `I/y`：先假装 I 永远不变，算一个"基准价"
- `(R-I)/[y×(1+y)^n]`：n 年后租金从 I 跳升到 R，这个"额外增量"折现回今天

**两种情形**：
```
R > I（欠租，rent below ERV）→ 第二项为正 → V > I/y（市场看好未来租金上涨）
R < I（超租，rent above ERV）→ 第二项为负 → V < I/y（租约到期后租金会下跌）
```

## Lecture 2 — Pro Forma & 快捷估值

---

### Q5：Pro Forma 里，为什么要"往后算一年"才能得到售价？

因为：

```
售价 = Year(n+1) NOI / Going-out Cap Rate
```

**你在第5年末卖楼，买家付的钱是基于"从第6年起"的收益。**

就像买债券，你付的价格是对未来所有现金流的折现——买家付钱是因为楼里未来还会产生租金，而这个租金从交割后第一年（即第6年）才开始流向买家。

**常见错误**：用 Year 5 NOI 算售价 → 售价偏低，IRR 偏低。

### Q6：转售价公式 `Year(n+1) NOI / Going-out Cap Rate` 是怎么推导的？各英文术语是什么意思？
### How is the Resale Value formula derived, and what do the terms mean?

**术语解释**：
- **NOI（Net Operating Income）**：净营业收入 = 租金收入 − 运营成本（不含融资成本）
- **Going-in Cap Rate**：买入时的资本化率——今天用市场价格和当前 NOI 反推出来的
- **Going-out Cap Rate**：卖出时的资本化率——将来买家用未来 NOI 和你开出的售价定出来的
- **Year(n+1)**：持有期结束后的**下一年**的 NOI

**公式推导**：

Cap Rate 的定义本身就是：
```
Cap Rate = NOI / 价格
```
反推得：
```
价格 = NOI / Cap Rate   ← 这就是转售价公式的来源
```

套入出售场景（第 n 年底卖楼）：
```
转售价 = Year(n+1) NOI / Going-out Cap Rate
```

**为什么用 Year(n+1) 而非 Year(n)？**

你在第 n 年底卖楼，买家从第 n+1 年才开始收到租金——买家付的价格是对**未来**现金流的折现，起点是第 n+1 年，不是已经收完的第 n 年。用 Year(n) NOI 算售价会系统性低估售价，IRR 也会偏低。

**为什么 Going-out ≥ Going-in（一般原则）？**

| 原因 | 解释 |
|------|------|
| 楼龄变老 | 设备磨损、维修成本上升，买家要求更高回报 |
| 租约剩余年限变短 | 更快面临重新招租不确定性 |
| 时间越远不确定性越大 | 买家折现率更高 → Cap Rate 更高 → 售价更低 |

Going-out 越高 → 转售价越低。这也是为什么考试中 Going-out 假设对整体 IRR 影响很大——它直接压缩或放大最终变现收益。

### Q7：租免期（Rent-Free）和超额租金（Overage Rent）在 Pro Forma 里怎么算？


**租免期（Rent-Free）**：那几期 NOI 直接设为 0（即使物理上在使用）
```
例：2年租免，第1-2年 NOI = 0；第3年起才开始收租
```

**超额租金（Overage Rent）**：在基础租金之上，按营业额的一定百分比收取
```
例：超额条款 = 租户营业额的 5%
    租户营业额 = £200,000
    超额租金 = £200,000 × 5% = £10,000（加在该年基础租金之上）
```

**两者共存情况**：租免期内超额租金也不收（整年 NOI = 0）

### Q8：Pro Forma 中哪五个"红旗"说明这笔交易好得离谱？
### What are the 5 Red Flags that signal a Pro Forma is too good to be true?

五个红旗都是同一个模式——要么**低估成本**，要么**高估收入**，要么**高估售价**：

**1. 历史成本计算土地（低估开发成本）**

土地成本应用**当前市场价（机会成本）**，而非当年买入价：
```
开发商2005年买地 £100万 → 现在市价 £500万
若 Pro Forma 用 £100万 → 成本低估 £400万 → 利润虚高
```
正确做法：土地 = 今天能卖出的价格（即使不打算卖）。

**2. 租金增长率 > 费用增长率（不合理假设）**

运营费用（维修、保险、管理费）和租金同受通胀驱动，增长率应接近：
```
若假设：租金 +5%/年，费用 +1%/年
→ 10年后 NOI 被严重高估
合理：两者均设为通胀率（如 2–3%）
```

**3. 空置率假设过低**

商业地产正常空置率 5–10%，假设 0% 则每年 NOI 虚增：
```
空置率 0%（假设）vs 7%（现实）
→ 每年 NOI 差 7% → 30年复利误差极大
```

**4. 退出 Cap Rate 过低**

Going-out Cap Rate 越低 → 售价越高（Price = NOI / Cap Rate）：
```
Going-out = 4%（激进）→ 售价虚高 → IRR 虚高
Going-out = 7%（保守）→ 售价真实 → IRR 真实
```
一般原则：Going-out ≥ Going-in（楼龄增加、不确定性更大）。若反之，需极强理由。

**5. 低估销售费用**

卖楼需支付经纪费、法律费、税费，通常占售价 1–3%：
```
售价 £1,000万，销售费用 2% = £20万
若忽略 → 净收益高估 £20万
```

**考试如何用**：看到 IRR 异常高的交易，检查这五项——哪里"便宜"了，哪里就是作弊点。

## Lecture 3 — 风险与杠杆

---

### Q9：WACC 公式为什么只是"近似值"，而不是精确答案？

**WACC 公式**：
```
rE = rP + (rP - rD) × D/E
```

**近似的根本原因：D/E 在整个持有期会一直变化。**

| 变化来源 | 影响 |
|---------|------|
| 物业升值 | E（股权价值）增加 → D/E 下降 |
| CPM 摊销 | D（债务余额）减少 → D/E 下降 |
| 物业贬值 | E 减少 → D/E 上升 |

WACC 公式只用"初始 D/E"——这意味着它假设整个持有期 D/E 保持不变，而实际不可能。

**全 DCF 方法（精确）**：逐年计算贷款利息/还款对现金流的影响，不假设 D/E 不变，所以更准确。

**考试中**：WACC ≈ DCF，但永远不等于，误差来自"D/E 随时间漂移"——这是必须能解释的。

### Q10：什么是正杠杆 vs 负杠杆？用什么条件判断？

```
正杠杆（Positive Leverage）：rP > rD  → 杠杆放大收益
负杠杆（Negative Leverage）：rP < rD  → 杠杆放大损失
```

**直觉**：借钱的成本是 rD。如果你用这笔钱能赚到 rP > rD，差价归你，而且放大了（因为借来的钱也在为你工作）。如果 rP < rD，每一块借来的钱都在亏损。

**2013年考题经典例子**：
- 基准情景：rP = 11.44%，rD = 10% → 正杠杆，IRR 从 11.44% 升到 16.47%
- 悲观情景（租金下降）：rP = 4.73%，rD = 10% → 负杠杆，IRR 从 4.73% 跌到 **-18.60%**

**BEIR（盈亏平衡利率）**= rP（无杠杆 IRR）。在 BEIR 借款无效，略低于才有利，切勿贴着 BEIR 借（市场波动可能让 rP 跌破 rD）。

### Q11：标准差（SD）和变异系数（CoV）在投资分析中代表什么？

```
E(IRR) = Σ Pᵢ × IRRᵢ           ← 加权平均回报（期望值）
Var = Σ Pᵢ × (IRRᵢ - E(IRR))²  ← 分散程度
SD = √Var                        ← 标准差（风险绝对量）
CoV = SD / E(IRR)                ← 变异系数（每单位回报承担多少风险）
```

**杠杆对三个指标的影响**：
- 杠杆 → E(IRR) 上升（好）
- 杠杆 → SD 上升（坏）
- 杠杆 → CoV 通常上升（每单位回报风险更大）

**结论**：杠杆没有"免费午餐"，永远是收益和风险的同步放大。

## Lecture 4 — 开发融资

---

### Q12：Turnkey / Forward Fund / Forward Commit 三种合同有什么区别？谁承担建设风险？

| 合同类型 | 投资者何时介入 | 建设风险归谁 | 投资者回报 |
|---------|-------------|-------------|----------|
| **Turnkey（交钥匙）** | 竣工后才付款 | **开发商**全部承担 | 较低（安全有价） |
| **Forward Fund（前向资金）** | 建设阶段分期出资 | **开发商+投资者**共担 | 中等 |
| **Forward Commit（前向承诺）** | 承诺竣工后购买，现在不出钱 | **开发商**承担 | 较高（投资者风险较低但时间不确定）|

**核心逻辑**：
- 投资者介入越早 → 承担越多建设风险 → 理论上回报越高
- Turnkey 最安全但利润最低（好东西都被开发商吃了）
- Forward Fund 是风险收益的中间地带，常见于大型机构投资

### Q13："Build at 10, Sell at 8" 法则如何快速判断开发可行性？

```
"Build to a 10"：稳定 NOI / 总开发成本 = 10%（建造端）
"Sell to an 8" ：稳定 NOI / 预期售价    = 8%（出售端）
```

**快速利润计算**：
```
利润 = 售价 - 开发成本
     = NOI/8% - NOI/10%
     = NOI × (12.5 - 10) = NOI × 2.5
     → 利润率约 25%（相对开发成本）
```

**反向运用（考试常考）**：给定开发成本，算"需要多高的租金才能达到 10%"：
```
所需 NOI = 总开发成本 × 10%
所需租金 = (所需 NOI + 运营成本) / (出租率 × 得房率)
```

## Lecture 5 — 私募股权

---

### Q14：IRR Lookback（回溯条款）是什么？LP 为什么会接受这个条款？

**标准分配（无 Lookback）**：
```
出售时，给 LP 保证 10% IRR 后，剩余按 50/50 分给 LP 和 GP
→ LP 拿超出 10% 的一半；GP 也拿一半
```

**有 Lookback 条款**：
```
出售时，给 LP 恰好达到 10% IRR；之后 100% 归 GP
→ LP 只拿 10%；GP 拿所有超额
```

**LP 为什么接受这个条款？**
- LP 的 10% 目标 IRR **保证性更强**（GP 用超额受益来换取对 LP 的 10% 承诺）
- LP 相当于买了"保底"换来的"上行封顶"
- 对风险厌恶型机构投资者（如养老金）：稳定的 10% 比不确定的 12-15% 更重要

**2020年考题数据**：
- 无 Lookback：LP IRR = 12.47%，GP IRR = 15.94%
- 有 Lookback：LP IRR = 10%，GP IRR = **20.27%**（GP 用上行换LP安全感）

---

### Q15：Deal-by-Deal vs Whole Fund 有什么区别？对 GP 的激励如何不同？

**Deal-by-Deal（逐笔交易结算）**：
```
每笔交易单独计算 Promote，不管整个基金表现
→ 若前3笔赚钱，GP 拿到 Promote；后2笔亏损，GP 不用吐出来
→ GP 激励：尽快做成赚钱的交易，不需要担心基金整体
```

**Whole Fund（整体基金结算）**：
```
所有交易合并计算，只有整体达到 Pref 后才给 GP Promote
→ GP 必须把亏损弥补后才能拿到超额
→ GP 激励：更谨慎，更长期，更关注整体组合质量
```

| | Deal-by-Deal | Whole Fund |
|--|-------------|-----------|
| LP 保护 | 较弱（GP 可能先吃好的） | 较强 |
| GP 激励 | 短期 | 长期 |
| 主要地区 | 美国市场常见 | 英国/欧洲市场更普遍 |

## Lecture 6 — 抵押贷款基础

---

### Q16：为什么 CAM 利率 < CPM 利率 < IO 利率？（银行视角）

**核心：银行收回资金越快，风险越低，利率越低。**

```
CAM（固定摊销）：每期都还大量本金，余额迅速下降
               → 银行风险敞口快速收缩
               → 利率最低

CPM（等额还款）：早期主要还利息，本金还得慢（前置利息）
               → 银行风险敞口下降缓慢
               → 利率中等

IO（纯利息）：  整个贷款期本金一分不还
               → 银行承担全部期限风险
               → 利率最高
```

**直觉比喻**：
- CAM = 贷款给有稳定收入且每月还很多本金的人（安全）
- IO = 贷款给20年只付利息、到期一次性还清的人（风险高）

---

### Q17：折扣点（Discount Points）为什么会让提前还款的成本更高？

**折扣点的本质**：一次性前期费用（sunk cost），分摊到整个贷款期限里。

```
有效成本 ≈ 合同利率 + 折扣点费用/贷款期限（年）
```

**持有期越短，折扣点摊薄越少：**

| 持有期 | 折扣点影响（2%折扣点，750,000贷款） |
|--------|--------------------------------|
| 25年（满期） | 每年约 600 → 有效成本轻微提升 |
| 10年 | 每年 1500 → 有效成本明显提升 |
| 5年 | 每年 3000 → 有效成本显著提升 |

**2025年考题验证**：
- 满期持有：有效成本 5.006%
- 10年还清：有效成本 5.0431%
- 5年还清：有效成本 5.1397%

**银行为什么喜欢折扣点？**
- 提前还款多发生在利率下降时（借款人去找更便宜的贷款）
- 银行被迫以更低利率重新放贷 → 损失收益
- 折扣点 = 提前还款的"违约金" + 筛选不打算挪位置的稳定客户

## Lecture 7 — 抵押贷款进阶

---

### Q18：ARM（可调利率贷款）为什么能提供比 FRM（固定利率贷款）更低的初始利率？

**本质：银行让出利率，换走了利率风险。**

```
FRM（固定利率）：
  银行锁定给你低利率 → 若市场利率上涨，银行亏损
  → 银行提前收取"利率风险溢价"→ 利率更高

ARM（可调利率）：
  利率跟随市场变化 → 市场涨了，你多付；市场跌了，你少付
  → 银行不承担利率风险 → 不需要收溢价 → 利率更低
```

**风险分配**：
- FRM：借款人承担"月供固定但实际购买力变化"的风险；银行承担"利率上升损失"
- ARM：借款人承担"利率上涨月供增加"的风险；银行只承担信用风险

**ARM 适合谁？**
- 短期内打算还清/出售房产（3-5年内）
- 相信利率未来会下降
- 收入有望增长能对冲月供上涨

---

### Q19：提前还款在 ARM 贷款中有时反而更便宜，这是为什么？

**正常情况**（无折扣点或固定利率）：提前还款 → 有效成本更高（折扣点摊薄期短）

**ARM 反常情况**：如果后期利率大幅提升，提前还款意味着**避开了高利率段**，实际上降低了整体成本。

**例（2018年考题）**：
```
ARM 利率：前2年 6% → 中2年 4% → 之后 8%
带3个折扣点

持有到期 IRR = 7.31%
第4年末还清 IRR = 5.94%（更低！）
```

→ 第4年末还清，避开了 8% 的高利率段，节省的利息 > 折扣点摊薄代价

**考试注意**：遇到 ARM 提前还款题，先不要假设"提前还款一定成本更高"，算完再下结论！

### Q20：通胀倾斜问题（Inflation Tilt）是什么？为什么是"非线性"的？

**问题根源**：CPM 的名义月供是固定的，但通胀侵蚀实际购买力。

**Fisher 方程**：
```
(1 + 名义利率) = (1 + 实际利率) × (1 + 通胀率)
如：实际 4% + 通胀 6% → 名义 ≈ 10.24%（不是简单相加！）
```

**倾斜效应**：
```
零通胀时月供 = £286（假设）
6% 通胀时月供 = £537（高 87.5%！）

但是：
10年后，通胀侵蚀了名义月供的实际价值
→ 早期月供实际购买力极高（负担重）
→ 后期月供实际购买力下降（负担轻）
```

**非线性的含义**：通胀率增加一倍，早期还款负担不止增加一倍——因为名义利率受通胀和实际利率的乘法影响（而非加法）。

**GPM 如何解决**：设计早期月供低、后期月供高，匹配实际还款能力和通胀侵蚀规律。

## Lecture 9 — 证券化与 REITs

### Q21：REITs 如何绕过直接房地产投资的三大风险？

| 直接投资风险 | REITs 如何解决 |
|------------|--------------|
| **流动性差**（变现困难）| REIT 股票在交易所交易，随时可卖 |
| **不可分割性**（£100万起步）| 可以买 1 股 REIT（£几十）|
| **信息不对称**（买卖双方信息差巨大）| 上市公司强制披露，信息透明 |

**REITs 的核心约束**：
- 必须分配 90%+ 的应税收益（限制留存利润再投资）
- 资产主要为房地产（不能随意多元化）
- 受监管，杠杆有上限

**价值创造策略**：
- 主动资产管理（提高出租率、优化租户结构）
- 收购低估资产后提升 NOI
- 发行股票/债券时机优化（低成本融资）
- NAV（Net Asset Value）折溢价管理

---

### Q22：MBS 证券化的 Cash Programme vs Swap Programme 区别是什么？

```
Cash Programme（现金方案）：
  贷款机构 → 出售给 GSE（获得现金）→ GSE 池化发行 MBS → 出售给投资者
  GSE 承担信用风险，收取 G-Fee（担保费）

Swap Programme（互换方案）：
  贷款机构 → 将贷款"互换"给 GSE → GSE 返回 MBS 证券（而非现金）
  贷款机构自己出售 MBS 给投资者
  GSE 仍承担信用风险，收取 G-Fee
```

**关键区别**：
- Cash: 贷款机构立即获得现金
- Swap: 贷款机构获得证券，自己承担卖出 MBS 的市场时机风险

---

### Q23：CMO（分层债券）vs Pass-Through（过手证券）的风险分配有何不同？

**Pass-Through（过手证券）**：
```
所有投资者按比例持有底层贷款池
提前还款风险 = 平均分配给所有投资者
→ 简单透明，但无法满足不同风险偏好的投资者
```

**CMO（担保抵押贷款债券）**：
```
将贷款池分成多个"层（Tranche）"：
Senior（优先层） → 最先收到还款，最后承担损失 → 最安全
Mezzanine（中间层） → 中等风险
Junior/Equity（次级层）→ 最后收到还款，最先承担损失 → 最高风险
```

**风险转移逻辑**：
- 高等级机构（养老金）买 Senior → 低风险低收益
- 风险偏好高的对冲基金买 Junior → 高风险高收益（或损失）
- 总风险不变，只是**重新分配**了谁先亏、谁后亏
