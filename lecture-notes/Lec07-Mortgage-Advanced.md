## 1. 提前还款罚金 Prepayment Penalties（延续 Lecture 6）

### 为什么贷款机构讨厌提前还款？

**贷款机构的困境：**
- 你以 8% 利率借出 £100 万，30 年
- 5 年后，市场利率跌到 5%，借款人全额提前还款
- 你拿回 £95 万（剩余余额），必须以 **5%** 再投出去
- 这 3% 的利差损失 × 剩余 25 年 = 巨大的利息损失

**解决方案：提前还款罚金（Prepayment Penalty）**

```
常见方式：还款时额外支付剩余余额的 X%（如 3%）

例：5年后提前还款，余额 £950,698，3% 罚金：
  罚金 = 950,698 × 3% = 28,521

计算实际 YTM：
CF0 = -990,000（扣折扣点后到手）
C01~C59 = 7,337.65（月供）
C60 = 7,337.65 + 950,698.25 + 28,521 = 986,557
IRR CPT → 年化 ≈ 8.72%（vs 无罚金时的 8.25%）
```

### 收益率维持条款 Yield Maintenance

**更精确的保护方式（商业地产常用）：**

```
借款人要提前还款时，需要支付补偿金，使贷款方的总收益
等同于"持有到期"的情况。

补偿金 = PV（原合同剩余利息 - 国债利率对应的替代利息）
        以国债利率为折现率

即：补偿贷款方因市场利率下降而损失的利息差
```

> **直觉**：你借钱时约定 8%，提前还款让贷款方只能以 5% 再借出去。Yield Maintenance 要求你补偿这 3% 的差额。

## 2. 通胀对抵押贷款的影响 Inflation & Mortgage Payments ⭐

### Fisher 方程：名义利率与实际利率

**通胀会影响利率！** 贷款方需要保护自己的实际购买力，所以在利率中加入通胀补偿。

```
Fisher 方程（精确版）：
(1 + 名义利率) = (1 + 实际利率) × (1 + 通胀率)

简化版（通胀较低时的近似）：
名义利率 ≈ 实际利率 + 通胀率
```

**数字例：实际利率 4%，通胀率 6%**
```
名义利率 = (1 + 4%) × (1 + 6%) - 1
         = 1.04 × 1.06 - 1
         = 1.1024 - 1
         = 10.24%（不是简单的 4+6=10%！）
```

### 通胀对月供的影响

**无通胀（实际利率 4%）：**
```
£60,000 CPM，30年，利率 4%
月供 PMT = £286.45
```

**有通胀（6%）：**
```
名义利率 = 10.24%
月供 PMT = £537.21（高出无通胀情景 87.5%！）
```

**时间维度分析：**

| 时间段 | 名义月供 | 实际月供（通胀调整后） |
|--------|---------|---------------------|
| 初期 | £537.21 | 极重（通胀未来才侵蚀） |
| 10年后 | £537.21（固定） | 较轻（通胀已侵蚀） |
| 30年后 | £537.21（固定） | 很轻（30年通胀大幅侵蚀名义值） |

**结论：** 高通胀时，固定名义月供的**初期实际负担极重**，后期逐渐变轻。

---

### "倾斜问题" Tilt Problem（通胀导致的还款前置问题）

**问题的本质：**
```
高通胀 → 高名义利率 → 高名义月供
→ 早期实际还款负担非常重
→ 越来越多的潜在借款人无法满足贷款资格（收入/月供 比率不达标）
→ 有效需求减少，购房者减少

随时间推移：
通胀逐渐侵蚀名义月供的实际价值
→ 后期还款变得越来越轻松
→ 但很多人在初期就被"筛掉"了
```

**图形直觉：** 在月供图上，高通胀情景的名义月供线（平）远高于无通胀情景（也平）；但实际月供线（通胀调整后）是从高到低的斜线，最终汇聚。问题是**初期的高实际负担**。

**倾斜问题的非线性：**
- 小通胀（如 2%）→ 小"倾斜"，轻微增加早期负担
- 大通胀（如 10%+）→ 极大"倾斜"，很多人直接买不起房

**如何解决倾斜问题？** 两种主要方法：
1. 渐进还款抵押贷款 GPM（早期低月供，后期高月供）
2. 可调利率抵押贷款 ARM（利率随通胀调整）

## 3. 渐进还款抵押贷款 GPM (Graduated Payment Mortgage)

### 设计思路

**GPM 的核心逻辑：** 与其让借款人承受高通胀期的前置还款重压，不如允许早期月供更低，后期逐步增加。

```
CPM（固定月供）：高通胀 → 月供 537.21（始终不变，早期痛苦）
GPM（渐进月供）：
  第1年：474.83（远低于 CPM！）
  第2年：510.44（×1.075）
  第3年：548.72（×1.075）
  第4年：589.87（×1.075）
  第5年：634.11（×1.075）
  第6-30年：681.67（稳定在高位，高于 CPM 的 617.17）
```

（上例使用 £60,000，12% 利率，每年 7.5% 递增，持续 5 年后固定）

### 负摊销机制（GPM 的数学）

**第1个月：**
```
月利率 = 12% / 12 = 1%
应付利息 = 60,000 × 1% = 600
实际月供 = 474.83
本金偿还（AMORT）= 474.83 - 600 = -125.17（负数！）
→ 这 125.17 没还，加进余额 → 负摊销
```

**第 1 年末：**
```
余额 BAL = 61,587（> 60,000！余额在增加！）
```

**计算器操作（求第1年末余额）：**
```
2nd [RESET] ENTER
2nd [P/Y] 12 ENTER CE/C
12 N                    → 12个月（第1年）
12 I/Y                  → 年利率 12%
60000 +/- PV            → 贷款额
474.83 PMT              → GPM 第1年月供
CPT FV                  → 显示 61,587.47（第1年末余额）
```

> **关键**：AMORT 工作表不能用于 GPM（因为月供不固定），必须用 **FV 键**手动计算各时点余额。

### GPM 月供的数学推导

**核心问题：** GPM 第 1 年月供怎么确定？

**思路：** 把整个 GPM 当作一个现金流，其 PV = 贷款额。

```
PV = 贷款额 = PV(第1-12期) + PV(第13-24期) + ... + PV(第61-360期)

其中：
  第 k 年的月供 = PMT₁ × (1+g)^(k-1)
  g = 7.5% 年增长率

公式：
PV = PMT₁ × [∑ 各年段折现因子]

第1年段（月1-12）：PMT₁ × [1-(1+i/12)^{-12}] / (i/12)
第2年段（月13-24）：PMT₁ × (1+g) × [1-(1+i/12)^{-12}] / (i/12) × (1+i/12)^{-12}
...以此类推
第6-30年段（月61-360）：PMT₁ × (1+g)^5 × [1-(1+i/12)^{-300}] / (i/12) × (1+i/12)^{-60}

反解 PMT₁ → 即第1年月供
```

（具体计算建议参考 B&F 教材，考试一般给数据而非推导）

### GPM 验证（用计算器确认利率正确）

```
CF Worksheet（验证 IRR = 12%）：
CF0 = -60,000
C01 = 474.83；F01 = 12
C02 = 510.44；F02 = 12
C03 = 548.72；F03 = 12
C04 = 589.87；F04 = 12
C05 = 634.11；F05 = 12
C06 = 681.67；F06 = 300
IRR CPT → 1%/月 = 12%/年 ✓
```

### GPM 的利弊权衡

**对借款人的优点：**
- 早期月供低，更容易"资格达标"（qualify）
- 适合预期收入增长的年轻借款人（收入也会随时间提高）

**对借款人的缺点：**
- 总利息支出更多（因为余额先增后减，早期利滚利）
- 后期月供高于 CPM，若收入没有按预期增长，后期压力大

**对贷款方的风险：**
- 初期负摊销 → 贷款余额**超过物业价值**（如果房价没涨）
- 抵押品保障变弱 → GPM 通常利率比 CPM 稍高（风险补偿）
- "安全边际"（Margin of Safety）在初期很小或负数

## 4. 可调利率抵押贷款 ARM (Adjustable Rate Mortgage)

### 为什么需要 ARM？

**固定利率贷款（FRM）的问题：**
- 贷款方承担全部利率风险：若利率大幅上升，银行的存款成本升高，但贷款收入固定，出现"倒挂"
- 为补偿这一风险，贷款方在固定利率上加了"利率风险溢价"→ FRM 利率更高

**ARM 的解决方案：** 让借款人分担部分利率风险，作为交换，贷款方提供更低的初始利率（"诱惑利率" Teaser Rate）。

### ARM 关键特征

```
基准利率（Index）：
  ARM 利率 = Index + Margin（加成）
  Index 常见：LIBOR（已废弃，现用 SOFR）、1年期国债利率

加成 Margin：固定数值，通常 2–3%

调整频率 Adjustment Interval：
  每年调整一次（1/1 ARM）
  前5年固定，之后每年调整（5/1 ARM）
  前3年固定，之后每年调整（3/1 ARM）

利率上限（Rate Caps）：
  Period Cap（单次调整上限）：如每次最多变动 ±2%
  Life Cap（整个贷款期总变动上限）：如全程不超过 ±6%

还款上限（Payment Caps）：
  限制每次调整时月供最大增幅（如 2%）
  注意：Payment Cap 可能导致负摊销！
```

### ARM 月供计算（每期重新计算）

**原则：** 每次调整利率后，用**当前余额**和**当前利率**，重新计算剩余期限的 CPM 月供。

**例题：** 借 £4M，2年期，2% Margin，无利率上限，年度调整

```
初始 Index = 26%，利率 = 26% + 2% = 28%

Year 1 月供：
  如同 £4M，2年期，28% 年利率，年度复利的 CPM
  PMT₁ = 4M × 28% / [1 - (1.28)^{-2}]
       = 1.12 / [1 - 0.610]
       = 1.12 / 0.390
       = £2.8744M
```

**Year 1 末余额：**
```
  如同以 28% 利率、2年期 CPM、付了一期 2.8744 后的余额
  BAL₁ = 2.8744 / 1.28 = £2.2456M（即剩余1年还款的现值）
```

**假设 Year 2 Index 变为 18%，新利率 = 18% + 2% = 20%**

```
Year 2 月供（重新计算）：
  如同 £2.2456M，1年期，20% 年利率的 CPM
  PMT₂ = 2.2456 × 20% / [1 - (1.20)^{-1}]
       = 0.44912 / (1 - 0.8333)
       = 0.44912 / 0.1667
       = £2.6947M
```

### ARM 有效成本计算

**无限制 ARM（继续上例）：**
```
借款 £4M，Year 1 还 2.8744，Year 2 还 2.6947

计算 IRR（有效年利率）：
4 = 2.8744/(1+IRR) + 2.6947/(1+IRR)²
IRR = 25.53%（低于 28% 的 Year 1 利率，因 Year 2 利率更低）
```

**利率风险溢价 = FRM 利率 - ARM 预期 IRR**

---

### ARM 利率上限（Rate Caps）的影响

**例：** 同上，但增加 2% 的年度利率上限（Period Cap）

```
Year 1：同前，利率 = 28%，PMT₁ = 2.8744

假设 Year 2 开始时 Index = 31%（上涨了 5%）
无上限情况：新利率 = 31% + 2% = 33%
有 2% 上限：利率最多从 28% 变到 30%（只能涨 2%）

Year 2 月供（受限利率 30%）：
  如同 £2.2456M，1年期，30% 利率
  PMT₂ = 2.2456 × 30% / [1 - 1.30^{-1}]
       = 0.67368 / (1 - 0.7692)
       = 0.67368 / 0.2308
       = £2.9193M
```

**对借款人：** 利率上限保护了借款人，防止利率大幅上升导致月供猛增
**对贷款方：** 利率上限是给借款人的"期权"，相当于贷款方承担了部分利率风险，故 ARM 有上限时利率一般比无上限时稍高

---

### ARM 还款上限（Payment Caps）的影响与负摊销风险

**例：** 借 £4M，2年期，2% Margin，**2% 还款上限（Payment Cap）**

```
Year 1：利率 28%，PMT₁ = 2.8744（同前）
Year 1 末余额 = 2.2456（同前）

假设 Year 2 开始时 Index = 38%，新利率 = 40%

无限制的 Year 2 月供 = 2.2456 × 1.40 = 3.1439（相对年 1 增长 9.4%）
但有 2% Payment Cap！
实际 Year 2 月供 = 2.8744 × 1.02 = 2.9319（封顶）
```

**问题：负摊销！**
```
Year 2 应付利息 = 2.2456 × 40% = 0.8982（年度） → 月化...
实际还款 2.9319，其中利息部分 = 2.2456 × 0.40 = 0.8982
本金偿还 = 2.9319 - 2.2456×0.40 = 2.9319 - 0.8982 = 2.0337

Year 2 末余额 = 2.2456 - 2.0337 = 0.2119（仍有余额！）
```

**贷款到期时还有余额 £0.2119M 未还！** 借款人和贷款方需要协商延长贷款期限或一次性还清。

> **重要提示**：ARM 的还款上限可能导致负摊销——这是次贷危机中部分问题贷款的特征之一（Option ARM：允许借款人选择还款额，最低额度低于利息 → 严重负摊销）。

## 5. 固定利率 vs 可调利率：谁应该选哪种？

### 从借款人角度

| | 固定利率 FRM | 可调利率 ARM |
|--|------------|------------|
| 利率风险 | **借款人承担**（锁定高利率，无法受益于利率下降） | **贷款方分担**（利率升时月供升，利率降时月供降） |
| 初期月供 | 较高（包含利率风险溢价） | 较低（Teaser Rate 吸引人） |
| 可预测性 | 高（月供30年不变） | 低（每年可能变化） |
| 适合场景 | 长期持有（10年+）、利率上行预期 | 短期持有（5年内卖掉）、利率下行预期 |
| 通胀保护 | 差（通胀高时名义月供不变但实际负担重） | 好（利率随通胀调整） |

### 从贷款方角度

- **FRM**：贷款方承担全部利率风险，但可预测现金流，适合需要稳定资产的机构（如养老金）
- **ARM**：贷款方将利率风险部分转移给借款人，降低自身利率敏感性

## 6. 有效抵押贷款成本汇总 Effective Cost Summary

```
实际借款成本（YTM）> 合同利率

因为以下因素增加成本：
1. 折扣点（Discount Points）：减少了到手金额 → YTM > 合同利率
2. 其他手续费（Origination Fees）：评估费、律师费等 → 同上
3. 提前还款罚金（Prepayment Penalties）：提前还款时额外支出 → YTM更高
4. 提前还款（即使无罚金）→ 固定前期费用摊薄期变短 → YTM更高
```

**规律：**
```
持有期越短 → 实际成本越高
（固定成本：折扣点等，在更短时间内摊销 → 年化成本更高）
```

## 考试高频考点 Exam Focus Points

> **历年真题分析 Based on Past Exams 2013–2025**

1. **ARM 分段计算（最高频计算题）** ★★★★★ — 2013, 2014, 2015, 2016, 2017, 2018, 2019, 2021, 2022, 2023, 2024, 2025（几乎每年）
   - 方法：每段视为独立 CPM → 段末用 AMORT 求余额作下段初始本金
   - 整体 IRR：CF Worksheet 分段输入不同 PMT 和期数
   - 注意 CF0 = -(贷款额 - 折扣点)（有折扣点时）

2. **ARM + 折扣点的实际成本** ★★★★★ — 2016, 2017, 2018, 2019, 2024, 2025
   - 含折扣点时：CF0 = -(贷款额 - 折扣点)；最后一期加剩余余额
   - ⚠️ **反常情况**：若后期利率很高，提前还款反而比持有到期成本更低（2016, 2018 曾出现）

3. **ARM 与 FRM 的风险分配 FRM vs ARM** ★★★ — 2022, 2023
   - FRM（固定利率）：银行承担利率风险，月供不变但实际价值随通胀侵蚀
   - ARM（可调利率）：借款人承担利率风险，初始月供低（teaser rate）

4. **GPM 渐进还款贷款计算** ★★★ — 2015, 2017, 2019, 2020
   - 分段 DCF 反解初始 PMT：`PV = PMT × [PV因子₁] + PMT×(1+g) × [PV因子₂] + ...`
   - 提前还款余额**不能**用 AMORT，必须用 FV 键手动计算（负摊销期间）

5. **通胀倾斜问题 Inflation Tilt + Fisher 方程** ★★★ — 2023（明确考察）
   - Fisher: `(1 + r_nom) = (1 + r_real) × (1 + π)`
   - 高通胀 → 名义月供大幅增加（"tilt"）→ 早期还款负担极重，借款人难以资格达标
   - GPM 是解决方案之一（早期低月供，后期高月供）

6. **负摊销 ARM（Teaser Rate）** ★★★ — 2019, 2024
   - 初期只付利息（甚至低于利息）→ 余额增加 → 重置后月供大幅跳升
   - 需用 FV 键而非 AMORT 计算重置时点的余额
