## 1. 为什么需要私募股权基金？ Why PE Funds?

**问题背景：** 我们前几讲谈到，房地产投资需要大量资金，且不可分割（Indivisibility）。那如果：
- 投资者资金不足以买下一栋楼怎么办？
- 想限制杠杆比例，但单靠自有资金又不够怎么办？
- 想分散到多栋物业，降低集中风险怎么办？

**解决方案：私募股权基金（Private Equity Fund）**

```
从多个投资者那里募集资金（如大学基金、养老金、高净值个人）
→ 汇集成一个大资金池
→ 由专业基金管理人进行投资决策
→ 按约定规则分配收益
```

PE 基金的核心价值：
1. **汇集资本**：突破单个投资者资金上限
2. **专业管理**：GP 有行业经验和交易资源
3. **风险分散**：跨多个物业投资，降低集中风险
4. **杠杆优化**：基金层面统一管理债务

## 2. 私募股权基金结构 PE Fund Structure

### 有限合伙制（Limited Partnership, LP Structure）

这是房地产私募基金最常见的法律结构：

```
┌─────────────────────────────────────┐
│     投资者（LP, Limited Partners）     │
│  养老金 / 大学基金 / 高净值个人        │
│  出资约 90% 的总资金                  │
│  分配约 80% 的总利润                  │
└───────────────┬─────────────────────┘
                │ 资金
                ▼
┌─────────────────────────────────────┐
│           基金（Fund）               │
│       有限合伙实体（LP Entity）       │
└───────────────┬─────────────────────┘
                │ 投资决策
                ▼
┌─────────────────────────────────────┐
│    基金管理人/赞助商（GP, General     │
│    Partner / Sponsor）               │
│  Blackstone, Cerberus, 专业 RE 机构  │
│  出资约 10% 的总资金                  │
│  分配约 20% 的总利润（含 Promote）    │
└───────────────┬─────────────────────┘
                │
                ▼
         物业投资（Properties）
```

### LP（有限合伙人）vs GP（普通合伙人）详细对比

| | LP（有限合伙人/投资者） | GP（普通合伙人/管理人） |
|--|----------------------|----------------------|
| 角色 | **被动**出资方 | **主动**管理方，日常决策 |
| 出资比例（典型） | 90% 的总资本 | 10% 的总资本 |
| 利润分配（典型） | 优先获得固定回报，然后 80% | 20%（Promote/Carry，超额奖励） |
| 法律责任 | **有限责任**：最多损失自己出的钱 | **无限责任**（通常为每个基金设单独 GP 实体，限制跨基金风险敞口） |
| 决策参与 | 极少（防止变成 GP，承担无限责任） | 完全控制 |
| 典型投资者 | 大学捐赠基金、养老金、家族办公室 | Blackstone, Carlyle, Apollo等 |
| 目标 | 稳定长期回报，风险有限 | 超额回报，获得 Promote 奖励 |

**为什么 GP 要自己也出钱（10%）？** 这是**利益对齐（Alignment of Interests）**机制。如果 GP 不出钱，他可能会冒过大风险（反正输的是别人的钱）。让 GP 自掏腰包，确保他也会谨慎管理。

## 3. 关键术语详解 Key Terms

### Preferred Return（"Pref"，优先回报率）

**含义：** LP 在 GP 分得任何超额利润之前，必须先达到的最低回报率（如 9% 年化）。

**类比：** 就像债权人优先于股东——LP 先拿到"保底收益"，剩余的才与 GP 分。

**为什么设置 Pref？**
- 保护 LP，确保管理人只在创造了足够价值后才能分利润
- 降低 LP 承担的风险（相当于"最低保障"）

### Promote / Carried Interest（利润分成 / 附带权益）

**含义：** 超过 Pref 的利润中，GP 可以获得的超额份额（通常为 20%）。

**类比：** 就像基金经理的"绩效费"——只有帮 LP 赚到足够多，自己才能拿到奖励。这是 GP 努力工作的核心激励。

**Promote 的重要意义：**
- GP 出了 10% 资金，却分 20% 超额利润 → 实际上是在拿超额部分的 10% 额外奖励
- 只有 IRR 超过 Pref，Promote 才起作用 → 激励 GP 追求高回报

### Waterfall（瀑布式分配）

**含义：** 规定利润如何按优先级"逐层流下"的分配规则，就像水从高处瀑布式往下流：

```
第1层：LP 先获得 Pref（如 9%）
  ↓
第2层：GP 获得 Pref（有时，按出资比例）
  ↓
第3层：超额利润按约定比例分（如 80% LP，20% GP）
  ↓
（出售时）
第4层：先还 LP 本金 → 还 GP 本金
  ↓
第5层：确保 LP 达到目标 IRR
  ↓
第6层：剩余全给 GP
```

### Pari Passu（同等比例）

**含义：** 按各方出资比例等比分配，没有优先级之分。相对少见（通常只用于分配本金部分）。

## 4. 简单瀑布分配例题 Simple Waterfall Example

### 例 1：Prushansky Pension（LP: $90M）vs Bhavik Fund（GP: $10M）

总投资 $100M，总利润 $25M，Pref = 9%

#### Option 1：超过 Pref 后再按 80/20 分

**原则：** 先所有人按比例拿到 9% 的 Pref，再把剩余利润 80% 给"出钱方"（LP+GP 按比例），20% 给 GP（Promote）

```
Step 1：总 Pref = 100 × 9% = $9M
  Prushansky（LP）获得：9M × (90/100) = $8.1M
  Bhavik（GP）获得：9M × (10/100) = $0.9M

Step 2：剩余利润 = 25 - 9 = $16M
  80%（"出钱方"按比例分）= 16 × 80% = $12.8M
    Prushansky 获得：12.8 × 90% = $11.52M
    Bhavik 获得：12.8 × 10% = $1.28M
  
  20%（GP 的 Promote）= 16 × 20% = $3.2M → 全给 Bhavik

汇总：
  Prushansky 总计 = 8.1 + 11.52 = $19.62M（占总利润78.5%）
  Bhavik 总计 = 0.9 + 1.28 + 3.2 = $5.38M（占总利润21.5%）
```

**Bhavik 出了 10% 的钱，拿到了 21.5% 的利润！** 这就是 Promote 的力量。

#### Option 2：先确保 LP 拿到 9% Pref，然后整体按 80/20 分

**原则：** 整笔利润直接按 80/20 分，但确保 LP 先拿到 9% Pref（不细分 Pref 阶段）

```
总利润 $25M：
  80%（"出钱方"）= $25M × 80% = $20M
    Prushansky：20M × 90% = $18M
    Bhavik：20M × 10% = $2M
  
  20%（GP Promote）= $5M → 全给 Bhavik

验证 Prushansky 回报率：
  $18M 利润 / $90M 投资 = 20% ✓（远超 9% Pref，LP 满意）

汇总：
  Prushansky 总计 = $18M（占利润72%）
  Bhavik 总计 = 2 + 5 = $7M（占利润28%）
```

**结论：Option 2 中，GP 在利润好时分到更多（$7M vs $5.38M）。** 两种模式各有适用场景，需要在基金文件中明确约定。

## 5. 复杂案例：ICI vs PDI 联合投资

**结构：** ICI（LP: $45M）与 PDI（GP: $5M）合作，总投资 $50M

### 现金流（经营期，年 1–5）

| 年 | 可分配现金流 |
|----|-----------|
| 1 | $1M |
| 2 | $2M |
| 3 | $5M |
| 4 | $6M |
| 5 | $6.5M |
| 年 5 末 | 出售：$75M |

### 瀑布分配规则（逐条解读）

```
规则 1：ICI（LP）先获得 5% 非累积优先回报（Non-Cumulative Pref）
  ↓
规则 2：PDI（GP）获得其 $5M 的 5% 非累积优先回报
  ↓
规则 3：超额经营现金流按 50/50 分（ICI 和 PDI 各半）
  ↓
规则 4（出售时）：先还 ICI 全部本金 $45M
  ↓
规则 5（出售时）：再还 PDI 全部本金 $5M
  ↓
规则 6（出售时）：额外补足 ICI 达到 12% IRR
  ↓
规则 7：剩余按 50/50 分
```

**关键概念解释：**

**非累积（Non-Cumulative）Pref：**
- 当年现金流不足支付 Pref → 不补；当年欠多少就欠多少，不累积到后年
- 对比：**累积（Cumulative）Pref** → 欠款累积并按利息滚动，后年必须补足（对 LP 更有保障）

### 经营期现金流分配

```
ICI 的 Pref = $45M × 5% = $2.25M/年
PDI 的 Pref = $5M × 5% = $0.25M/年
```

| 年 | 可分配 | ICI Pref | PDI Pref | 余额 | ICI(50%) | PDI(50%) | ICI 总计 | PDI 总计 |
|----|--------|---------|---------|------|---------|---------|---------|---------|
| 1 | $1M | $1M | 0 | 0 | 0 | 0 | $1M | 0 |
| 2 | $2M | $2M | 0 | 0 | 0 | 0 | $2M | 0 |
| 3 | $5M | $2.25M | $0.25M | $2.5M | $1.25M | $1.25M | $3.5M | $1.5M |
| 4 | $6M | $2.25M | $0.25M | $3.5M | $1.75M | $1.75M | $4M | $2M |
| 5 | $6.5M | $2.25M | $0.25M | $4M | $2M | $2M | $4.25M | $2.25M |

**注意：第 1–2 年现金流不足以支付双方 Pref（ICI Pref 为 $2.25M，但只有 $1M 和 $2M），所以 PDI 这两年什么都拿不到。**

### 出售现金流分配（IRR 偏好机制）

**总出售额 $75M 的分配：**

Step 1：还 ICI 本金 $45M
Step 2：还 PDI 本金 $5M（剩余 $75 - 45 = $30M，扣本金后 $25M）
Step 3：额外支付 ICI，使其整体 IRR 达到 12%

```
计算 ICI 需要多少出售收益才能达到 12% IRR：
ICI 的经营期现金流：1, 2, 3.5, 4, 4.25（见上表）
初始投资：-$45M
IRR = 12% 时，需要第5年末收到：$66,051,668（含本金）

即：第5年末 ICI 总收益 = $66,051,668
其中：经营现金流 = $4.25M（已分配）
    本金 = $45M（先还）
    额外支付 = 66,051,668 - 4,250,000 - 45,000,000 = $16,801,668
```

Step 4：剩余 = 75 - 45 - 5 - 16,801,668 = $8,198,332，按 50/50 分
  → ICI 额外获得 $4,099,166，PDI 获得 $4,099,166（但这已包含在 PDI 本金回收中）

**最终 IRR：**
```
ICI IRR = 12%（刚好达到目标）
PDI IRR = 26.64%（大幅超额！）
```

## 6. IRR 回溯机制 IRR Lookback

### 什么是 Lookback？

**普通 IRR 偏好（Preference）：** 从出售收益中，先给 ICI 补足到 12% IRR 所需金额，剩余再按 50/50 分。

**IRR 回溯（Lookback）：** 重新分配所有出售收益，使得**ICI 的整体 IRR 刚好等于 12%，多出的全给 PDI**。

**区别：**
- 偏好机制：ICI 先拿 12% IRR 所需，剩余 50/50 → ICI 实际可能拿到 12%，PDI 拿到更多超额
- 回溯机制：确保 ICI 刚好 12%，PDI 拿走全部超额 → 更有利于 GP（PDI）

```
Lookback 结果：
ICI IRR = 12%（恰好）
PDI IRR = 32.94%（比偏好模式的 26.64% 更高！）
```

### 为什么 ICI 会接受 Lookback？

- 如果最终表现好，ICI 的 Lookback 与偏好结果差不多
- Lookback 通常配合**更高的 Pref** 或**更多保护条款**
- LP 接受 Lookback 换取 GP 更努力工作（PDI 的激励更强）

## 7. 非累积 vs 累积优先回报 Non-Cumulative vs Cumulative Pref

| | 非累积 Non-Cumulative | 累积 Cumulative |
|--|---------------------|----------------|
| 当期不足 | 不补偿，直接损失 | 累积到后续年份，加利息补足 |
| 对 LP | 风险较高（好年份少拿也没有追索） | **更有保障**（必须补足所有欠款） |
| 对 GP | 负担较轻 | 负担重（好年才能拿 Promote） |
| 常见程度 | 相对常见（早期现金流不确定时） | 较少见，但对 LP 更友好 |

**例：如果 ICI 有累积 Pref：**
- 第 1 年：欠 ICI Pref = $2.25 - $1 = $1.25M → 累积
- 第 2 年：欠 = $2.25 - $2 = $0.25M → 累积，外加第1年欠款的利息
- 第 3 年：先补偿历史欠款（$1.25 + $0.25 + 利息），然后才分当期超额

## 8. 基金运作方式补充 Additional Fund Mechanics

### Deal-by-Deal vs Whole Fund 结算

**Deal-by-Deal（逐笔交易结算 Promote）：**
- 每笔投资单独计算 Promote
- GP 可以在早期成功交易上就拿到 Promote，即使后期有亏损
- 常见于美国；**对 GP 更有利，对 LP 风险更高**

**Whole Fund（全基金合并结算 Promote）：**
- 所有交易合并后才计算总体 Promote
- 必须整个基金所有投资平均下来超过 Pref，GP 才能拿 Promote
- 常见于英国；**对 LP 更有保障**

### 资金"按需拨付"（Capital Calls / Drawdown）

LP 签署投资承诺后，不是一次性全部交钱：
- GP 按投资需要，逐次"Call"（拨款通知）LP 提供资金
- LP 需在通知后几天内转账（通常 5–10 个工作日）
- 对 LP 有利：资金在 Call 之前可以用于其他投资

## 考试高频考点 Exam Focus Points

> **历年真题分析 Based on Past Exams 2013–2025**

1. **LP/GP 结构、角色与风险分配** ★★★★ — 2019, 2020, 2022, 2025
   - LP（有限合伙人）：被动出资，优先获得固定回报（Pref），损失有限
   - GP（普通合伙人）：主动管理，承担更高风险，换取 Promote 超额收益
   - 2022、2025 均考理论题讨论 LP/GP 风险如何分配

2. **Waterfall 瀑布分配计算** ★★★★ — 2020（完整计算题含 Lookback）
   - 顺序：①LP 获 Pref → ②GP 获 Pref → ③按比例（如 50/50）分超额
   - 出售时：先还 LP 本金 → 还 GP 本金 → 再按目标 IRR 分 → 剩余再分

3. **Promote 激励机制** ★★★★ — 2025（理论题重点）
   - Promote = GP 超过 Pref 后获得的超额利润分成（如 20%）
   - 核心逻辑：GP 只有在创造超额回报后才获得额外收益 → 激励努力管理

4. **Deal-by-Deal vs Whole Fund 结算方式** ★★★ — 2025（新考点）
   - Deal-by-deal：每笔交易独立结算 Promote（美国常见）→ GP 早期受益，LP 风险高
   - Whole Fund：所有交易合并计算 Promote（英国常见）→ LP 更有保障

5. **IRR Lookback（回溯条款）计算** ★★★ — 2020（完整计算）
   - 从售价中先补足 LP 达到目标 IRR（如 10%）所需金额
   - 方法：用 PV 公式反算 x，使得 LP 现金流 IRR 恰好等于目标；剩余全给 GP
   - 2020 具体题目：IRR preference（12%）条件下，剩余 50/50 vs lookback 时 GP IRR 从 15.9% 跳到 20.3%
